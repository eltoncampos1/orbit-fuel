## Codebase Patterns
- Calculator functions use `floor/1` (Elixir built-in) for integer truncation
- Travel domain lives in `lib/orbita_fuel/travel/`
- `total_fuel/2` uses reverse accumulation: process steps in reverse, each step's mass = base_mass + accumulated fuel from later steps (rocket equation)
- Step uses `Ecto.Enum` for :action and :planet fields within an `embedded_schema`
- Step update events: use `<form phx-change="update_step">` with `name="step_id"` hidden input (NOT `name="id"` — causes LiveView warning)
- phx-value-* attributes are NOT sent in render_change for select elements; only phx-click events get phx-value-*
